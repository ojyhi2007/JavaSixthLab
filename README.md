## Задание 1 — Аннотация @Invoke

<details>
<summary>Подробнее об задании</summary>

### Аннотация Invoke

### Описание:
Разработана аннотация **@Invoke**, которая используется для пометки методов, которые должны вызываться автоматически.

Главная цель задания — показать, как с помощью **Reflection API** можно находить методы с определённой аннотацией и вызывать их во время выполнения программы.

### Логика решения:
- Аннотация применяется только к **методам**.
- Она доступна **во время выполнения программы** (`RetentionPolicy.RUNTIME`).
- Аннотация **не имеет свойств**.
- Создан класс `InvokeExample`, содержащий методы, один из которых помечен аннотацией `@Invoke`.
- Класс `InvokeProcessor` с помощью Reflection:
  - получает все методы класса,
  - проверяет наличие аннотации `@Invoke`,
  - автоматически вызывает найденные методы.

### Пример:
- Создаётся объект `InvokeExample`.
- Обработчик `InvokeProcessor.process()` находит метод `start()`, помеченный `@Invoke`.
- Метод автоматически вызывается и изменяет состояние объекта.

</details>



## Задание 2 — Аннотация @Default

<details>
<summary>Подробнее об задании</summary>

### Аннотация Default

### Описание:
Разработана аннотация **@Default**, которая позволяет задать **тип по умолчанию** для класса или поля.

Главная цель задания — показать, как аннотации могут хранить информацию о типах и как эту информацию можно извлечь с помощью Reflection.

### Логика решения:
- Аннотация может применяться к:
  - **классу**
  - **полю**
- Аннотация доступна **во время выполнения программы**.
- Она имеет обязательное свойство `value` типа `Class<?>`.
- Создан класс `DefaultExample`, который:
  - помечен аннотацией `@Default` на уровне класса,
  - содержит поле с аннотацией `@Default`.
- Класс `DefaultProcessor`:
  - считывает аннотацию у класса,
  - считывает аннотации у полей,
  - выводит имя указанного класса по умолчанию.

### Пример:
- Класс может иметь аннотацию `@Default(String.class)`.
- Поле может иметь аннотацию `@Default(Integer.class)`.
- Обработчик выводит соответствующие типы.

</details>



## Задание 3 — Аннотация @ToString

<details>
<summary>Подробнее об задании</summary>

### Аннотация ToString

### Описание:
Разработана аннотация **@ToString**, которая управляет тем, какие поля объекта должны участвовать в его строковом представлении.

Главная цель задания — создать механизм генерации строки объекта на основе аннотаций.

### Логика решения:
- Аннотация может применяться к:
  - **классу**
  - **полю**
- Аннотация имеет параметр `Mode`, который может принимать значения:
  - `YES`
  - `NO`
- Значение по умолчанию — `YES`.
- Если поле помечено `@ToString(Mode.NO)`, оно не включается в строковое представление объекта.
- Класс `ToStringBuilder`:
  - получает список полей объекта,
  - проверяет наличие аннотации,
  - формирует строку, включающую только разрешённые поля.

### Пример:
- Класс содержит поля `name`, `age`, `password`.
- Поле `password` помечено `@ToString(Mode.NO)`.
- При формировании строки пароль не выводится.

</details>

## Задание 2 — Тестирование

<details>
<summary>Подробнее об задании</summary>

Вторая часть лабораторной работы посвящена **тестированию разработанных аннотаций** с использованием фреймворка **JUnit 5**.

Главная цель — проверить корректность работы обработчиков аннотаций и убедиться, что программа правильно обрабатывает как корректные, так и ошибочные случаи.

В рамках варианта 9 реализованы тесты для аннотаций **@Validate** и **@Two**.

---

### Тестирование аннотации `@Validate`

Создан тестовый класс **ValidateProcessorTest**, который проверяет работу обработчика `ValidateProcessor`.

#### Проверяется:

- корректное извлечение списка классов из аннотации `@Validate`
- выбрасывание исключения `IllegalArgumentException`, если массив классов пуст

#### Логика теста:

1. Для класса `ValidateExample` проверяется, что метод  
   `ValidateProcessor.getValidatedClasses()` возвращает массив классов:
   - `String`
   - `Integer`
   - `Double`

2. Для класса `EmptyValidateExample` проверяется, что при пустом массиве
   выбрасывается исключение `IllegalArgumentException`.

#### Пример:

```java
assertThrows(IllegalArgumentException.class, () -> {
    ValidateProcessor.getValidatedClasses(EmptyValidateExample.class);
});
